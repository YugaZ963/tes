Dalam GitHub, "branch" merujuk pada salinan dari kode sumber atau proyek yang ada di repositori. Branch digunakan untuk mengembangkan fitur baru, memperbaiki kesalahan, atau membuat perubahan lain pada kode sumber tanpa memengaruhi branch utama atau "master".
Setiap repositori GitHub memiliki setidaknya satu branch default yang disebut "master", namun pengguna dapat membuat branch tambahan sesuai kebutuhan. Ketika pengguna membuat branch baru, GitHub akan membuat salinan dari branch utama saat itu, dan pengguna dapat melakukan perubahan pada salinan tersebut tanpa memengaruhi branch utama.
